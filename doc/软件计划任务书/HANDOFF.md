## 已记录的关键架构结论

- 旧方案：B 端运行 LLM-Wiki 和 OpenCode，并通过 frp 发布完整 OpenCode 页面给 C 端。
- 旧方案问题：模型配置归属不清、B 端 Agent 能力暴露、C 端访客行为运行在 B 端环境。
- 新方案：B 端只发布 LLM-Wiki API；S 端提供 BearFRP/frps；C 端本地运行 OpenCode，并通过 llm-wiki MCP 访问远程知识库。
- 新方案价值：B/S/C 三端职责清晰，C 端自持模型配置，B 端不暴露完整 OpenCode，安全边界更可控。
