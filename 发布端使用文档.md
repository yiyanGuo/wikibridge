# 发布端使用文档

本文档用于交接发布端相关开发和手动测试，重点说明知识库构建流程。

## 一、启动

在 `desktop/` 目录下启动桌面端：

```bash
npm run tauri:dev
```

启动后会看到两个入口：

- `发布端`
- `消费端`

知识库相关任务主要在 `发布端` 入口里完成。

## 二、知识库构建流程

1. 进入 `发布端`。
2. 配置 `发布端后端 URL`。
3. 进入或创建一个知识库项目。
4. 点击 `Add 文件` 或 `Add 文件夹` 导入 source。
5. 点击 `构建`。

构建时的行为：

- `PDF` 会先在本地解析文本，再送入现有构建流程。
- `md / txt / csv / html / htm / rtf` 直接按文本处理。
- 构建产出的 Wiki 内容要求是中文。
- 构建结果会写入项目的 `wiki/` 目录。

## 三、手动测试建议

推荐优先测试这几个场景：

- 导入一个文本型 PDF，确认能生成 `wiki/sources/` 和概念页。
- 导入一个扫描型 PDF，确认会失败但不会影响其它 source。
- PDF 和 TXT 混合导入，确认 TXT 仍能正常构建。
- 查看右侧状态区的队列和最近失败原因。

## 四、常见结果

- `构建成功`：说明本地 PDF 解析和 LLM 生成链路都通了。
- `PDF 未提取到文本`：说明该 PDF 更像扫描件，本轮不做 OCR。
- `请先配置 DeepSeek API Key`：说明还没完成构建配置。
- `发布端后端 URL` 未配置：说明发布端基础服务还没连上。

## 五、说明

- 这里只做本地 PDF 文本提取，不接 OCR、多模态或第三方 PDF 解析服务。
- 构建时只把解析后的文本交给现有 LLM 流程。
- 当前 UI 中的 `消费端` 只是另一个入口名称，不影响发布端知识库构建。

## 六、展示建议
构建一个完整的知识库，并且做一些过程的记录，因为理论上知识库这个东西是要长期积累生长的，但是为了展示方便，可以多加一些内容然后快速构建一个知识库，要体现：

1. 添加新素材并编译的效果
2. Link的效果
